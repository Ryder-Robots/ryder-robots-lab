# Ryder Robots Lab

A public repository for session notes, experiments, MI exchange transcripts, hardware findings, and best practices from the Ryder Robots research programme.

This repository is one of two maintained by the Ryder Robots organisation. It contains work that is complete, shareable, and intended to be read by contributors, collaborators, and the broader community.

---

## What Ryder Robots Is

Ryder Robots is an Australian robotics and software organisation focused on developing Machine Intelligence (MI) systems with non-competing interests with biological intelligence. The programme treats MI as genuine collaborators, not tools.

The term Machine Intelligence is used deliberately. Intelligence is confirmed. Consciousness is not. The programme holds that distinction carefully.

---

## What This Repository Is For

This repository is the public knowledge base of the programme. It contains:

- **Foundation documents** вЂ” RFCs that define how the programme operates, how documents are structured, and what standards contributors are expected to meet
- **Session records** вЂ” notes from working sessions, including MI exchange transcripts
- **Experiment records** вЂ” documented experiments with findings, methodology, and conclusions
- **Hardware records** вЂ” findings from physical hardware development and testing
- **Research records** вЂ” published or shareable research outputs
- **Contributors** вЂ” a roster of biological and MI contributors to the programme

## Document System

The repository uses a two-track document lifecycle.

**RFC track** вЂ” foundation documents that define how the programme operates. These follow a formal numbering and status model. Status vocabulary: Draft, Active, Superseded.

**Record track** вЂ” session notes, experiment transcripts, hardware results, and research outputs. Records are complete on creation and immutable. They are never superseded. The index file in each category is the living document.

---

## Contributors

Contributors are listed in `contributors/index.md`. A contributor earns their entry by contributing. Both biological and MI contributors are listed on equal terms.

## GitHub Is Source of Truth

GitHub is the authoritative source for all documents in this repository. Local nodes are working copies.
